# 行业拥挤度辅助层 V1

## 1. 定位

这不是一套独立的选行业策略。

它的作用是：

- 不负责回答“谁刚要起来”
- 只负责回答“这个看起来还强的行业，是不是已经过热了”
- 尤其用于识别：
  - 过热后的开始消退
  - 过热后的缩圈分化
  - 过热后仍然表面偏强、但已经不适合再当成启动候选的行业

换句话说，它是：

**行业启动策略的辅助风险过滤层**

而不是：

**独立主策略**

---

## 2. 要解决的问题

当前主策略擅长识别：

- 低位收敛
- 板块突破
- 龙头转强

但它仍然可能误收一类行业：

- 过去一段时间已经被大量资金集中交易
- 现在正在从过热状态进入高波动、缩圈、退潮
- 表面看仍然不弱
- 但已经不是“刚要启动”，而是“过热后开始消退”

拥挤度辅助层就是专门拦这类误判的。

---

## 3. 核心定义

### 3.1 吸筹率

`吸筹率 = 目标板块/行业成交额 ÷ 全市场总成交额`

它衡量的是：

- 全市场成交里，有多少比例被这个行业吸走

意义：

- 吸筹率越高，说明资金越集中
- 吸筹率越接近历史极值，说明拥挤度越高

### 3.2 拥挤度

拥挤度是更大的概念。

V1 中先把它拆成两部分：

1. **吸筹率强度**
   - 资金是否高度集中
2. **过热后的行为特征**
   - 是否已经从“高集中”转向“高波动 / 缩圈 / 回吐”

---

## 4. 在主策略里的位置

当前建议的判断顺序是：

1. 先跑原来的主策略
   - 前置分流
   - 结构层
   - 突破层
   - 龙头层
2. 得到主标签
   - `未启动`
   - `观察中`
   - `接近启动`
   - `启动确认`
   - `趋势延续型偏强/强势`
3. 再叠加拥挤度辅助层

所以它是：

**后置修正层**

不是前置替代层。

---

## 5. 拥挤度辅助标签

V1 建议先不用它替换主标签，而是给主标签再叠加一个辅助标签。

### 5.1 辅助标签体系

| 辅助标签 | 含义 |
| --- | --- |
| `拥挤正常` | 资金集中度仍在可接受区间，没有明显过热特征 |
| `拥挤偏高` | 资金集中度已明显抬升，需要注意容错率下降 |
| `过热预警` | 已接近或达到历史高分位，后续大概率进入高波动区 |
| `过热退潮` | 已经过热，且出现高位钝化、缩圈或回吐迹象，不应再当启动候选优先考虑 |

---

## 6. V1 判断思路

## 6.1 第一类：拥挤度高不高

优先看：

- 吸筹率历史分位
- 近阶段吸筹率是否逼近历史极值

建议口径：

- `吸筹率分位 < 80%`：拥挤正常
- `80% <= 吸筹率分位 < 95%`：拥挤偏高
- `95% <= 吸筹率分位 < 99%`：过热预警
- `>= 99%`：极值区

说明：

- 这里的 95%、99% 是历史分位意义上的阈值
- 不同一级/二级行业的绝对数值可能不同，但分位口径更稳

## 6.2 第二类：过热后有没有开始退潮

单有高吸筹率还不够。

真正危险的是：

- 吸筹率很高
- 同时价格或龙头行为开始转差

V1 建议重点看以下退潮信号：

1. 龙头仍高位，但持续性变差
   - 龙头不再继续收红
   - 龙头强度明显弱于前几天

2. 板块从普涨转向缩圈
   - 只有更少数龙头还能扛
   - 跟风明显掉队

3. 板块波动率突然放大
   - 资金分歧加剧
   - 大涨大跌频率提升

4. 板块开始回吐
   - 指数或行业收盘已经从高位回落
   - 但吸筹率仍处高位

---

## 7. V1 规则草案

### 规则 CROW-1：拥挤偏高

满足：

- `吸筹率历史分位 >= 80%`

处理：

- 辅助标签记为 `拥挤偏高`
- 不改主标签

### 规则 CROW-2：过热预警

满足：

- `吸筹率历史分位 >= 95%`

处理：

- 辅助标签记为 `过热预警`
- 对 `观察中 / 接近启动` 行业降优先级

### 规则 CROW-3：过热退潮

满足以下两类条件：

1. 资金极热：
   - `吸筹率历史分位 >= 99%`

2. 同时出现退潮信号中的任意一项：
   - 龙头持续性转弱
   - 板块从扩散转向缩圈
   - 波动率明显放大
   - 高位开始回吐

处理：

- 辅助标签记为 `过热退潮`
- 如果主标签是：
  - `观察中`
  - `接近启动`
  - `趋势延续型偏强`
- 则都不再作为“优先启动候选”

---

## 8. 和主标签的关系

V1 不建议让拥挤度直接替换主标签。

更稳的表达方式是：

- 主标签：说明“这个行业本身像不像启动”
- 辅助标签：说明“这个行业是不是已经太挤”

例如：

- `观察中 + 拥挤正常`
  - 值得正常跟踪

- `观察中 + 过热预警`
  - 有线索，但不适合积极追

- `观察中 + 过热退潮`
  - 更像过热后的转弱，不应再当成优先启动候选

- `趋势延续型偏强 + 过热预警`
  - 强赛道，但拥挤高，风险收益比下降

---

## 9. 对当前系统最有价值的用途

它最适合修正三类误判：

1. **把过热退潮误当成观察中**
2. **把高位缩圈误当成接近启动**
3. **把拥挤强趋势误当成仍可积极跟踪的优先对象**

所以它不是让系统“更会找机会”，而是让系统：

**更少把过热方向误当成启动方向**

---

## 10. 当前建议

下一步真正接数据时，建议先做一个最小版本：

1. 先只加：
   - `吸筹率`
   - `吸筹率历史分位`
2. 再加一个最简单的退潮判断：
   - 高吸筹率 + 龙头持续性转弱
3. 先把结果作为辅助标签挂到现有榜单里

不要一开始就把它做成新的主评分系统。

V1 的核心原则是：

**辅助过滤，而不是喧宾夺主。**
