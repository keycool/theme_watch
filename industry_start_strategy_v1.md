# 行业启动策略 V1

## 1. 策略目标

这套策略只回答一个问题：

**某个行业是否正在从沉寂或整理阶段，进入值得重点跟踪的“启动阶段”。**

它的目标不是：

- 找出所有当前最强的行业
- 给所有上涨行业贴标签
- 替代趋势跟踪策略

换句话说，这是一套：

**启动识别策略**

而不是：

**强势行业总筛选策略**

---

## 2. 先讲边界

在这套策略里，最重要的不是“如何评分”，而是先分清：

1. 什么行业属于“启动识别对象”
2. 什么行业不属于“启动识别对象”

如果这一步没分清，后面就会把两类完全不同的行业混在一起：

- 一类是“刚从低位走出来”
- 一类是“已经走了很长趋势，还在继续强”

前者是本策略要抓的。

后者不是本策略的核心目标。

---

## 3. 判断顺序

这套策略的判断顺序固定如下：

### 第一步：先分流

先判断行业属于哪一类：

- `启动识别对象`
- `趋势延续对象`

### 第二步：只有在“启动识别对象”里，才继续细分

细分为：

- `未启动`
- `观察中`
- `接近启动`
- `启动确认`

### 第三步：如果不属于启动识别对象

则不再进入四档启动判断，而是直接标记为：

- `趋势延续型偏强`
- `趋势延续型强势`

---

## 4. 输出体系

在输出层面，这套策略分成两个口径：

1. `内部观察口径`
   用于研究、跟踪、预警，允许保留中间状态，例如 `观察中`、`接近启动`、`趋势延续型偏强`。
2. `严格确认口径`
   用于回答“这个行业是否已经完成启动确认”，这里必须坚持三条核心条件同时满足，不轻易给出“启动确认”。

也就是说：

- `观察中` 是内部预警，不等于行业已经启动
- `接近启动` 是内部候选，不等于行业已经确认
- 只有三条核心条件闭环，才进入 `启动确认`

## 4.3 拥挤度辅助层

在主策略之外，另保留一层：

- **拥挤度辅助层**

它不是用来替代主标签的，而是用来修正一类常见误判：

- 有些行业看起来还强
- 但其实已经处在高吸筹率、高拥挤度之后的缩圈、分化或回吐阶段
- 这类行业不应再被当成“刚要启动”的优先候选

因此当前推荐的理解顺序是：

1. 先跑主策略，得到：
   - `未启动`
   - `观察中`
   - `接近启动`
   - `启动确认`
   - `趋势延续型偏强/强势`
2. 再叠加拥挤度辅助标签：
   - `拥挤正常`
   - `拥挤偏高`
   - `过热预警`
   - `过热退潮`

也就是说：

- 主标签回答“像不像启动”
- 拥挤度辅助层回答“是不是已经太挤”

详见：
- [industry_crowding_filter_v1.md](/D:/CC/Industry%20Insight/industry_crowding_filter_v1.md)

## 4.1 启动识别体系

这是本策略的主输出体系。

| 结论 | 含义 |
| --- | --- |
| `未启动` | 板块仍处于弱势、震荡或下行状态，没有出现明确启动信号 |
| `观察中` | 板块或龙头出现局部转强，但条件不完整，暂不确认启动 |
| `接近启动` | 板块层与龙头层已有较强共振，只差一个关键确认条件 |
| `启动确认` | 必须满足三条核心条件同时成立：低位收敛结构、带量突破年线、标志性龙头显著强势且具备持续性 |

## 4.2 趋势延续体系

这不是看空标签，只是说明它们已经不属于“刚启动”的样本。

| 结论 | 含义 |
| --- | --- |
| `趋势延续型偏强` | 行业已脱离低位启动阶段，但趋势强度一般，偏震荡上行或中等偏强 |
| `趋势延续型强势` | 行业已运行较长强趋势，核心龙头位置高、强度高，不属于启动识别对象 |

---

## 5. 什么属于“启动识别对象”

一个行业要进入“启动识别主流程”，应尽量满足以下特征：

1. 过去一段时间并不是市场最强主线
2. 板块仍处于中低位区间，尚未明显脱离底部区域
3. 行业尚未走出持续很长时间的主升趋势
4. 当前更像“准备启动”或“刚开始启动”，而不是“已经涨了很久”

更直白地说：

**我们要找的是“刚要起来”的行业，而不是“已经飞了很久”的行业。**

---

## 6. 什么不属于“启动识别对象”

以下行业，不应继续进入 `未启动 / 观察中 / 接近启动 / 启动确认` 的判断链条：

1. 已经走出较长时间上升趋势的行业
2. 明显处于高位强势区间的行业
3. 核心龙头普遍已经站上中长期均线、并维持较长时间强势的行业
4. 近阶段本身就是市场主线或高热方向的行业

这类行业应归入：

- `趋势延续型偏强`
- `趋势延续型强势`

---

## 7. 前置分流规则

## 7.1 前置排除原则

在进入三层判断前，先做一次分流：

- 如果行业更像“低位整理后准备启动”，进入启动识别主流程
- 如果行业更像“已经走出较长趋势”，直接归入趋势延续体系

## 7.2 前置排除规则 P1：高位强趋势排除

如果行业满足以下特征中的任意两项，则不进入“启动识别”主流程：

- `当前收盘价 > 250日均线 * 1.15`
- `近120日涨幅分位 >= 80%`
- `当前收盘价 >= 近120日最高收盘价 * 0.95`
- 板块核心龙头大面积站上 `60日均线` 与 `250日均线`

说明：

- 这类行业通常已经不是“刚启动”
- 它们更适合做趋势跟踪，而不是做启动识别

## 7.3 分流后的处理

### 如果满足 P1

直接停止后续四档判断，并标记为：

- `趋势延续型偏强`
  适用于已经脱离低位，但整体强度不算极强的行业
- `趋势延续型强势`
  适用于已经走出明显主升趋势、核心龙头群体强的行业

### 如果不满足 P1

才进入后面的启动识别三层判断。

---

## 8. 启动识别三层结构

只有通过前置分流、被认定为“启动识别对象”的行业，才进入这一部分。

### 第一层：板块整理结构

目标：

- 判断行业是否经过一段整理
- 判断它是否具备“低位收敛”的启动前特征

### 第二层：板块突破行为

目标：

- 判断板块是否从整理状态中走出
- 判断突破是否有效、是否带量、是否有持续性

### 第三层：龙头确认

目标：

- 判断板块内部是否已经有代表性龙头率先转强
- 判断这种转强是否具有带动性和持续性

---

## 9. 启动识别规则定义

## 9.1 板块整理结构

说明：

- 当前 V1 主要使用日线量化代理来近似“低位收敛”
- 它能提供稳定、可批量运行的结构判断，但仍不完全等同于附件里更强调的周线图形结构
- 后续如果继续升级，这一层应逐步向“周线级别、高点降低、低点抬高、筹码沉淀”的图形语义靠拢

### 规则 A1：低位区间

定义：

- 观察窗口：近 120 个交易日
- 当前收盘价距离近 120 日最高点仍有明显距离

建议口径：

- `当前收盘价 <= 近120日最高收盘价 * 0.90`

用途：

- 避免把高位板块误判为“刚启动”

### 规则 A2：波动收敛

定义：

- 观察近 120 个交易日，拆成前 80 日和后 40 日
- 如果后 40 日振幅明显小于前 80 日，视为进入收敛整理

建议口径：

- `后40日振幅 <= 前80日振幅 * 0.90`

其中：

- `振幅 = (区间最高价 - 区间最低价) / 区间均价`

### 规则 A3：不再创新低

定义：

- 最近 40 个交易日不再跌破近 120 日最低点

建议口径：

- `近40日最低价 >= 近120日最低价 * 1.02`

### 规则 A4：阶段涨幅不居前

定义：

- 近 120 日涨幅分位不高

建议口径：

- `近120日涨幅分位 <= 50%`

### 板块整理结构判定

`A1 + A2 + A3 + A4` 同时满足，记为：

- `结构成立`

否则记为：

- `结构未成立`

---

## 9.2 板块突破行为

### 规则 B1：站上年线

定义：

- 板块收盘价站上 250 日均线

建议口径：

- `最新收盘价 >= 250日均线 * 1.03`

### 规则 B2：带量突破

定义：

- 板块成交额明显高于近期均值

建议口径：

- `最新成交额 >= 20日平均成交额 * 1.20`

### 规则 B3：突破后不回落

定义：

- 突破后至少连续 2 个交易日站在线上

建议口径：

- `最近2个交易日收盘价都在250日均线之上`

### 板块突破判定

- `突破出现`：满足 `B1 + B2`
- `突破确认`：满足 `B1 + B2 + B3`

---

## 9.3 龙头确认

### 规则 C1：龙头候选识别

定义：

- 在板块成分股里找最有代表性的龙头候选

建议口径：

- 默认按流通市值或总市值排序，取前 3 名作为龙头候选

### 规则 C2：龙头率先走强

定义：

- 龙头候选中，至少 1 只明显强于板块

建议口径：

满足以下任意一条即可：

- 当日涨幅 >= 7%
- 最近 5 日涨幅位于板块成分前 20%
- 当日涨停

说明：

- `涨停` 是更强的确认信号，但不是唯一硬门槛
- 对大市值行业龙头而言，连续强势、持续跑赢板块，很多时候比单日涨停更有代表性
- 本策略定位偏“观察与识别”，因此允许用 `7%大阳` 或 `5日强势` 提前捕捉龙头转强

### 规则 C3：龙头持续性

定义：

- 龙头不是一天脉冲，而是随后仍保持强势

建议口径：

满足以下任意一条即可：

- 次日收红
- 次日收盘高于前一日收盘
- 次日盘中不明显走弱，收盘跌幅绝对值小于 2%

说明：

- 这一条比“是否涨停”更重要
- 单日涨停如果次日迅速转弱，确认意义反而不如连续两天到三天的稳步走强

### 规则 C4：龙头趋势位置

定义：

- 龙头本身至少接近或站上中期均线

建议口径：

- `龙头最新价 > 60日均线`

加强口径：

- `龙头最新价 > 250日均线`

### 龙头确认判定

- `龙头转强`：满足 `C1 + C2`
- `龙头确认`：满足 `C1 + C2 + C3 + C4`

---

## 10. 启动识别中的最终决策规则

这一部分只适用于已经通过前置分流、被认定为“启动识别对象”的行业。

### `未启动`

满足以下任意一种：

- 板块整理结构未成立
- 板块突破未出现
- 龙头转强未出现

### `观察中`

满足以下任意一种：

- 板块结构成立，但突破未出现
- 板块未突破，但龙头先转强
- 板块突破出现，但龙头确认未完成

### `接近启动`

同时满足：

- 板块结构成立
- 板块突破出现
- 龙头转强出现

但以下至少一项未满足：

- 板块突破确认
- 龙头确认

### `启动确认`

同时满足：

- 板块结构成立
- 板块突破确认
- 龙头确认

## 11. 数据源分工建议

### 板块层

需要：

- 历史日线
- 历史成交额
- 250 日均线
- 20 日均额
- 120 日涨幅分位

适合的数据源：

- `Tushare sw_daily`
- 其他能稳定返回板块历史日线的接口

### 龙头层

需要：

- 板块成分股
- 个股最新行情
- 近 20 日强弱
- 换手率
- 流通市值
- 涨停/大涨识别

适合的数据源：

- `TDX MCP`
- `Tushare daily / daily_basic / stk_limit`

---

## 12. 当前项目阶段建议

当前最适合的实施顺序是：

1. 先把“启动识别”和“趋势延续”正式分开
2. 先验证少数典型行业样本
3. 先输出明确标签，不急着自动生成长文
4. 等板块层历史数据稳定后，再扩成全行业扫描

---

## 13. V1 的边界

这版策略故意保持保守，不解决以下问题：

- 不处理盘中实时信号
- 不处理多龙头并行的复杂情形
- 不处理题材股短期情绪爆发但板块指数未同步的特例
- 不做回测优化
- 不直接生成交易建议

V1 的任务只有一个：

**先把“启动型行业”和“趋势型行业”分清，再把启动阶段本身规则化、可复核化、可自动化。**

## 14. 2026-06-11 规则校准：避免把长期站上年线误判为启动

这次校准来自 `工程咨询服务Ⅱ(801726.SI)` 的复核。

该行业在旧规则下被误判为 `启动确认`，但复核后发现：

- 它不像典型的低位收敛结构，更像回落后的阶段整理。
- 它虽然仍在 250 日均线上方，但不是“新近带量突破年线”。
- 只检查“当前是否站上年线”会把长期已经在线上的行业误判为启动。

因此 V1 规则做以下收紧：

### 14.1 低位结构收紧

旧口径：

- `当前收盘价 <= 近120日最高收盘价 * 0.90`
- `近120日涨幅分位 <= 50%`

新口径：

- `当前收盘价 <= 近120日最高收盘价 * 0.85`
- `近120日涨幅分位 <= 40%`

这一步的目的，是减少“并不够低、只是回落整理”的行业被判成低位收敛。

### 14.2 突破年线必须是近期新突破

旧口径只检查：

- `最新收盘价 >= 250日均线 * 1.03`
- `最新成交额 >= 20日均额 * 1.20`
- `最近2日站上年线`

新口径增加：

- `above_ma250_3pct_streak <= 20`

含义是：

- 如果一个行业已经连续很久站在 `250日均线 * 1.03` 上方，它就不再属于“新近突破年线”。
- 这类行业更适合按趋势延续、震荡整理或再强化来理解，不应直接进入 `启动确认`。

### 14.3 对最终标签的影响

`启动确认` 仍然必须三条同时闭环：

- 低位收敛结构成立
- 近期带量突破年线成立
- 龙头显著强势并具备持续性

如果只有龙头强，而板块结构或近期突破不成立，应降为 `观察中`，不能给 `启动确认`。
