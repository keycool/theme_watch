# ETF 主题观察 SOP

## 1. 定位

这套主题观察系统不是替代申万二级全市场扫描，而是把“我正在关注的 ETF / 主题指数”映射到最值得观察的申万二级行业。

核心目的：
- 从用户关注入口出发，而不是盲扫全市场。
- 用日收益相关性确认 ETF 与申万二级的对应关系。
- 最终仍回到申万二级图形页，观察年线、收敛路径、拥挤度和龙头状态。

指标口径：
- 主指标负责判断 `未启动 / 观察中 / 接近启动 / 启动确认 / 趋势延续`。
- 主指标主要描述中长期结构，包括年线位置、低位收敛、突破行为和龙头确认。
- 辅助指标只负责提示 `拥挤 / 过热 / 波动 / 映射质量`，不单独决定启动。
- 辅助指标主要描述短期状态，包括吸筹率分位、波动率异常、拥挤度和数据映射质量。
- 主/辅分类不按一阶或二阶函数机械划分；即使某个主指标使用标准差表达，只要它服务于中长期结构判断，仍归入主指标。
- 均值回归模型只吸收方法，不照搬配置权重；启动阶段由主指标决定，辅助指标只做观察优先级和风险修正。
- 龙头确认使用“龙头群”而不是单一龙头：按申万二级成分总市值排序，优先覆盖前 `60%` 市值，最少 `3` 只、最多 `10` 只；第一龙头只用于展示和辅助解释。
- 龙头活跃允许用 `5%` 当日涨幅、`7%` 大阳或 `5日强势` 捕捉，不把涨停作为唯一条件；启动确认仍要求板块结构、年线突破和龙头群共同闭环。
- 图形观察同时保留 `MA60` 和 `MA250`：`MA60` 用于观察中期修复和季度线方向，`MA250` 仍是长期启动确认的核心年线。
- 启动观察信号允许前移到 `MA60`：若 `MA60` 曾明显低于 `MA120`、指数低位收敛并冲击/突破 `MA60`，可标为 `早期启动`；但真正的 `启动确认` 仍需等待年线和龙头条件闭环。
- 详细分层见 [indicator_hierarchy_v1.md](/D:/CC/Industry%20Insight/indicator_hierarchy_v1.md)。

### 1.1 原始策略中心思想

这套策略要寻找的不是当天涨幅最大的行业，也不是所有处于低位的行业，而是：

> 长期低位、冷门行业在完成筹码沉淀后，由增量资金推动趋势反转，并最终得到行业龙头和市场共识确认的启动过程。

完整观察链条是：

> 低位收敛蓄力 → 板块带量突破年线 → 标志性龙头持续上涨。

三个条件是层层递进、互相验证的串联关系，不是可以彼此替代的加分项：

- 低位收敛只说明行业具备潜在启动基础，不代表资金已经进场。
- 放量突破年线说明中长期趋势可能反转，但仍可能是假突破或短期脉冲。
- 龙头持续上涨用于确认行业逻辑已获得市场认可，而不是少量小票的局部异动。
- 只有三层同时闭环，才属于严格意义上的 `启动确认`。

因此，策略的核心任务是识别“刚要起来”的行业，并把“已经涨了很久”的强趋势行业分流为 `趋势延续`，避免追涨对象与启动对象混为一谈。

### 1.2 三个核心条件

#### 条件一：低位形成收敛图形结构

市场含义：

- 行业前期经过较长时间沉寂、下跌或冲高回落。
- 随着抛压释放和筹码交换，价格波动逐步收窄。
- 行业半年左右的累计表现明显落后于热门行业，仍处于相对低位。
- 理想图形是在周线级别看到高点逐渐降低、低点逐渐抬高，形成横盘或收敛整理。

这一层确认的是“筹码沉淀和低位蓄力”，不是单纯寻找跌幅最大的行业。持续阴跌、不断创新低或波动仍在扩大的行业，不应视为结构成立。

#### 条件二：板块整体带量突破年线

市场含义：

- 年线代表中长期趋势边界，板块指数突破年线意味着原有下跌趋势开始被扭转。
- 突破必须有明显增量成交配合，才能说明资金形成一致性进场。
- 收盘价应有效站上年线，原始观察口径以高于年线约 `3%` 为有效突破参考。
- 突破后还要观察能否站稳，避免把盘中冲高或单日脉冲当成趋势反转。

这一层是启动的资金触发信号。只有价格突破而没有成交量验证，不能直接认定行业启动。

#### 条件三：标志性龙头上涨并形成持续性

市场含义：

- 优先观察行业中市值和影响力靠前的标志性公司，而不是小市值股票的偶发异动。
- 原始严格口径强调市值排名前 `3` 的龙头中至少一只出现涨停。
- 龙头强势后，下一交易日继续收红或随后数日持续上涨，才说明上涨不是单日脉冲。
- 龙头持续走强意味着行业上涨逻辑得到更广泛资金认可，并可能吸引场外资金继续进入。

这一层是市场共识确认。板块上涨但核心龙头没有响应，或者只有边缘小票上涨，信号可信度应降低。

### 1.3 严格确认口径与当前监控口径

原始策略的严格口径是：

1. 周线或中长期级别形成清晰的低位收敛结构。
2. 板块指数放量并有效突破年线。
3. 行业标志性龙头出现强势上涨，并在后续交易日保持持续性。

当前系统为了提高日常扫描的覆盖率，使用了可计算的日线代理指标，并对部分触发条件进行了监控层面的放宽：

- 用近 `120` 日高低位置、前 `80` 日与后 `40` 日波动区间、近期是否停止创新低，以及行业涨幅排名，近似描述低位收敛。
- 用指数高于 `MA250` 约 `3%`、当日成交额达到 `20` 日均值的 `1.2` 倍、连续站稳及突破时效，描述带量突破年线。
- 龙头层使用覆盖行业前 `60%` 市值、最少 `3` 只且最多 `10` 只的龙头群；以当日明显上涨、近 `5` 日强势、次日延续和站上 `MA60` 等指标捕捉转强，不把涨停作为唯一触发条件。

这些放宽项服务于 `早期启动`、`观察中` 和 `接近启动` 等监控标签，不能改变以下解释原则：

- `MA60` 信号只能提前预警，不能替代 `MA250` 年线突破。
- 龙头群中出现一只活跃股票，只能说明局部转强，不能单独确认行业启动。
- 拥挤度、吸筹率和短期活跃度是辅助风险指标，不能替代三个核心条件。
- `启动确认` 必须由低位结构、年线突破和龙头持续性共同闭环。

## 3. 新增 ETF 的标准流程

### 第一步：确认基金日线

默认优先尝试：

```powershell
py .\build_theme_to_sw_l2_correlation.py --ts-code 代码.SH --source fund --output .\reports\theme_watch\correlations\theme_代码_to_sw_l2_correlation.csv
```

深市 ETF 使用 `.SZ`：

```powershell
py .\build_theme_to_sw_l2_correlation.py --ts-code 代码.SZ --source fund --output .\reports\theme_watch\correlations\theme_代码_to_sw_l2_correlation.csv
```

如果返回 0 行：
- 不硬替代。
- 先核对代码是否写错。
- 只有用户确认后，才用相近代码替换。

### 第二步：判断映射质量

看相关性表前 5 名：
- `>= 0.90`：高度纯映射，可以作为主观察对象。
- `0.70 - 0.90`：可用映射，但需要看是否分散到多个行业。
- `< 0.50`：低相关异常，先做校验页，不纳入正式主题组。

### 第三步：生成专题图形页

如果是独立主题：

```powershell
py .\build_sw_l2_topic_report.py --codes 申万代码1 申万代码2 申万代码3 --title "主题名 - 申万二级关联观察" --output .\reports\theme_watch\pages\theme_xxx_sw_l2_watch_report.html
```

如果是对照或异常：
- 文件名使用 `compare` 或 `check`。
- 文档中明确写出异常原因或对照目的。

## 4. 合并规则

可以合并：
- 两个 ETF 的前 5 个申万二级高度重叠。
- 核心申万二级相同，差异只是宽基/窄基。
- 行业归属一致，且用户观察目的相同。

不建议合并：
- 只是风格相近，但申万二级结构不同。
- 只是走势相关，但申万一级不同。
- 相关性来自行情扩散，而不是行业归属。

已合并案例：
- `159928 消费 ETF` + `512690 酒 ETF` → 消费酒主题组
- `512010 医药 ETF` + `512170 医疗 ETF` → 医药医疗主题组
- `515230 软件 ETF` + `159998 计算机 ETF` → 软件计算机主题组

已拆分案例：
- `512880 证券 ETF` 中的软件/计算机对象拆出，证券页只保留非银金融。
- `159870 化工 ETF` 从周期资源组拆出，单独作为基础化工对照。

## 5. 页面分类

### 正式主题页

用于日常观察。文件名一般是：
- `theme_xxx_sw_l2_watch_report.html`
- `theme_group_name_sw_l2_watch_report.html`

### 对照页

用于观察相邻但不应混入核心主题的对象。文件名一般包含：
- `compare`
- `对照`

### 校验页

用于相关性异常或数据口径待确认。文件名一般包含：
- `check`
- `低相关校验`
